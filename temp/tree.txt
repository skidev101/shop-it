====================================================
KEY FRONTEND NOTES
====================================================

CALLING YOUR EXPRESS BACKEND
  All backend calls live in lib/api/*.ts
  client.ts is the base fetch wrapper — set NEXT_PUBLIC_API_URL
  in .env.local to point at your Express server.
  React Query (TanStack Query) handles caching, refetching,
  and loading/error states throughout the app.

ROUTE GROUPS (no effect on URL)
  (marketing)  — homepage, landing pages
  (shop)       — products, cart, checkout, search
  (vendor)     — public vendor storefronts /[vendorSlug]
  (dashboard)  — protected vendor dashboard
  (account)    — buyer account: orders, wishlist
  (auth)       — sign in, sign up, onboarding

MULTIVENDOR CART PATTERN
  Cart items are keyed by vendorId in cart-store.ts.
  cart-summary.tsx renders totals broken down per vendor.
  checkout/shipping-options.tsx lets buyers pick shipping
  per vendor sub-order before paying.

RECOMMENDED FRONTEND STACK
  Framework   — Next.js 14+ (App Router)
  Styling     — Tailwind CSS + shadcn/ui
  State       — Zustand (cart, UI)
  Data fetch  — TanStack Query (React Query)
  Forms       — React Hook Form + Zod
  Payments    — Stripe.js + @stripe/react-stripe-js
  Auth        — Clerk or NextAuth (JWT from your Express API)
  Charts      — Recharts
  Search UI   — cmdk (command palette)
  Carousel    — Embla Carousel
